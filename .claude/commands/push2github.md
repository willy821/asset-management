# Push to GitHub

Perform a full GitHub publish cycle for this project: security scan, push code, update README, and update the repo About section.

## Steps

### 1. Security scan — block if secrets found
Before touching git, scan all tracked files for secrets or API keys. Check for:
- Strings matching patterns: `sk-`, `ghp_`, `xox`, `AKIA`, `AIza`, `Bearer `, `api_key`, `apikey`, `secret`, `password`, `token` (case-insensitive) inside `.html`, `.js`, `.css`, `.json`, `.md`, `.yml`, `.yaml` files.
- Any `.env` files that are not in `.gitignore`.
- Any hardcoded credentials that are not clearly placeholder values.

If any real secret is found, **stop immediately** and tell the user which file and line contains the issue. Do not proceed until resolved.

### 2. Ensure .gitignore exists and is correct
Check if `.gitignore` exists. If not, create one that ignores:
```
.env
*.env
.env.*
node_modules/
*.log
.DS_Store
Thumbs.db
```

### 3. Stage and push all changes
Using the git binary at `C:\Users\willy.teo\AppData\Local\Programs\Git\cmd\git.exe`:
- `git add .`
- `git commit -m "<auto-generated message describing what changed>"` — write a meaningful commit message based on what files changed.
- `git push origin main`

If there is nothing to commit, skip the commit step and still attempt the push.

### 4. Update README.md
Rewrite `README.md` to accurately reflect the current state of the project. Always include:
- Project name and one-line description
- Live site link: `https://willy821.github.io/asset-management`
- Features list (derive from current `index.html` sections)
- Project structure table (`index.html`, `styles.css`, `script.js`)
- How to run locally
- Contact email from `index.html` or `CLAUDE.md`

After writing, commit and push: `git commit -m "Update README"` then `git push origin main`.
